Plinko.js

White Paper Live Github

Most recently, I co-created a real-time, multiplayer, physics-based game that can be played in the browser using only JavaScript, requiring no special client-side plugins. I used Node.js and WebSockets to create a client-server networking model that can synchronize state generated by a physics engine across the internet to multiple clients in real time at 60fps. I built the client site and matchmaking system with React, and a custom animation renderer with HTML5 Canvas. To simulate a real-time, lag-free environment, I implemented client-side prediction algorithms and binary data serialization for transmitting application state.

Read more about the networking, state synchronization, and other challenges in our white paper.
